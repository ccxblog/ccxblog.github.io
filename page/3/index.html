异常

  • 什么是异常

    • 在实际工作中，遇到的事情不可能是完美的。比如：你写某个模块，用户输入不一定合法，你的程序打开某个文件，这个文件可能不存在或者文件格式不对，你要读取数据库的数据，数据可能是空的等。
    • 异常的英文 Exception , 意思是例外。遇到这些例外或者说是异常，怎么让程序做出合理的处理。而不至于让程序奔溃。
    • 异常在程序执行中，影响了程序的正常执行顺序。

  • 简单分类

    • 检查性异常：最具代表性的是用户错误或者问题引起的异常，这是程序无法预见的。例如要打开一个不存在的文件时候，一个异常就产生了，这些异常在编译时不能被简单的忽略。

    • 运行时异常：是可以被程序员避免的异常，与检查性异常相反，运行时异常可以在编译的时候被忽略。

    • 错误：错误不是异常，而是脱离程序员控制的问题。错误在代码中通常被忽略，列如当栈溢出时错误就发生了，它在编译的时候是检查不到的。

Java面向对象

发表于 2020-11-13 更新于 2020-11-14 分类于 java

面向对象思想

  • 物以类聚，分类的思维模式，思考问题首选解决问题需要哪些分类，然后对某个分类下的细节进行面向过程的思索。

  • 面向对象适合处理复杂的问题，适合处理需要多人协作的问题。

  • 面向对象的本质就是：以类的方式组织代码，以对象的组织（封装）数据。

  • 面向对象的三大特性：封装 ， 继承 ， 多态。

  • 抽象 : 只关注问题本身，去除和问题无关的部分。（例如 ： 学生成绩管理系统 ， 我们只在乎学生的学号，姓名，成绩等因素 ， 而学生的家庭住址，有多少零花钱是不需要关注的）

类跟对象的关系

  • 类是一种抽象的数据类型，它是对某一类事物整体的描述/定义，但是并不代表某一个具体的事物。

    • 动物，植物，手机，电脑……
    • Person类 ， Pet类 ， Car类等，这些类都是用来描述/定义某一类具体的事物应该具备的特点和行为。

  • 对象是抽象概念的具体实列

Java中的方法

什么是方法

  • System.out.println() 其中 System 是一个类 ， 而 out 是 System 的一个对象 ， println() 就是这个对象的方法。

  • Java方法是语句的集合 ， 它们在一起执行一个功能。

    • 方法是解决一类问题的步骤的有序集合。
    • 方法包含于类和对象中。
    • 方法在程序中被创建，在其它地方被引用
  • 设计方法的原则：设计方法的时候最好保存方法的原子性，就是一个方法只完成一个功能，这样有利于后期的扩展。

注释

Java中有三种注释

  • 单行注释 : // 注释一行文字

    public class Helloworld {
    public static void main(String[] args){
        // 单行注释
        // 输出一个Hello World
        System.out.println("hello world");
    }
}

  • 多行注释 ： /* 开始然后中间一段文本 */结束

    public class Helloworld {
    public static void main(String[] args){
        /* 注释
           注释
           注释
           注释
        */
        System.out.println("hello world");
    }
}
