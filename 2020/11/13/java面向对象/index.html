Java面向对象

发表于 2020-11-13 更新于 2020-11-14 分类于 java

面向对象思想

  • 物以类聚，分类的思维模式，思考问题首选解决问题需要哪些分类，然后对某个分类下的细节进行面向过程的思索。

  • 面向对象适合处理复杂的问题，适合处理需要多人协作的问题。

  • 面向对象的本质就是：以类的方式组织代码，以对象的组织（封装）数据。

  • 面向对象的三大特性：封装 ， 继承 ， 多态。

  • 抽象 : 只关注问题本身，去除和问题无关的部分。（例如 ： 学生成绩管理系统 ， 我们只在乎学生的学号，姓名，成绩等因素 ， 而学生的家庭住址，有多少零花钱是不需要关注的）

类跟对象的关系

  • 类是一种抽象的数据类型，它是对某一类事物整体的描述/定义，但是并不代表某一个具体的事物。

    • 动物，植物，手机，电脑……
    • Person类 ， Pet类 ， Car类等，这些类都是用来描述/定义某一类具体的事物应该具备的特点和行为。

  • 对象是抽象概念的具体实列

    • 张三就是人的一个实列，张三家里的旺财就是狗的一个实列。
    • 能够体现出特点，展现出功能的是具体的实列，而不是一个抽象的概念。

创建和初始化对象

  • 使用 new 关键字创建对象

    使用 new 关键字创建的时候，除了分配内存空间之外，还会给创建的对象进行默认的初始化以及对类中的构造器的使用。

    • 类中的构造器也称为构造方法，是在进行创建对象的时候必须调用的。并且构造器有以下两个特点：

      1. 必须和类的名字相同。
      2. 不能有返回类型 ， 也不能写 void。

    • 注意的是 在定义有参构造后 ， 必须显示的定义

class Person{
    String name;
    Person(){ name = "xiaomin" ;} // 无参构造函数
    Person(String s){ name = s; } // 有参构造函数
}

public class Application {
    public static void main(String[] args) {
        Person a = new Person();   //无参构造
        Person b = new Person("xiaohon");//有参构造
    }
}

封装

  • 该露的露，该藏的藏

    • 我们程序设计要求 “高内聚，低耦合”。高内聚就是类的内部数据操作细节自己完成，不允许外部干涉；低耦合：仅暴露少量的方法给外部使用。

  • 封装(数据的隐藏)

    • 通常，应该禁止直接访问一个对象中数据的实际表示，而应该通过操作接口来访问，这称为信息的隐藏。

    • 这样做的目的的可以在方法里面做出安全性的检查。

class Person{
    private int age; // 年龄
    public int  H; // 身高
    public void setAge(int age) {
        if(age > 100) return ; // 安全检查
        this.age = age;
    }
}
public class Application {
    public static void main(String[] args) {
        Person per = new Person();
        per.H = (-100); // 给身高赋值为 -100 ， 不能检查错误
        per.setAge(999); // 给年龄赋值 999 ， 可以判断检查合法 
    }
}

继承

  • 继承的本质是对某一批类的抽象，从而实现对现实世界更好的建模。

  • extends的意思是拓展。子类是父类的拓展。

  • Java里只有单继承，没有多继承。（一个儿子只能有一个爸爸，一个爸爸可以有多个儿子）

  • 继承是类与类的一种关系，除此之外类与类的关系还有依赖，组合，聚合等。

  • 继承关系的两个类，一个为子类（派生类），一个为父类（基类）。子类继承父类，使用关键字extends表示。

class Person{
    String name;
    int age;
    public void say(){
        System.out.println("say");
    }
    private void sayHello(){
        System.out.println("say , hellow");
    }
}
class Teacher extends Person{

}
public class Application {
    public static void main(String[] args) {
      Teacher tea = new Teacher();
      tea.say();
      // tea.sayHello() 不合法
    }
}

super 关键字

super可以理解为是指向自己超（父）类对象的一个指针，而这个超类指的是离自己最近的一个父类。

super也有三种用法：

1.普通的直接引用：与this类似，super相当于是指向当前对象的父类，这样就可以用super.xxx来引用父类的成员。

2.子类中的成员变量或方法与父类中的成员变量或方法同名。

3.引用构造函数：super（参数）：调用父类中的某一个构造函数（应该为构造函数中的第一条语句）。

class Person{
    Person(){}
    protected String name = "Person";
    public void print(){ System.out.println("Person"); }
}
class Teacher extends Person{
    Teacher(){
        super();
        System.out.println("Teacher");
        // super(); 不行只能放在第一条
    }
    private String name = "Teacher";
    public void print(){ System.out.println("Teacher"); }

    void setName(String name){
        System.out.println(name);       // 输出 tea
        System.out.println(this.name);  // 输出 Teacher
        System.out.println(super.name); // 输出 Person
    }
    void test(){
        print(); // 输出Teacher
        this.print(); // 输出 Teacher
        super.print(); // 输出 Person
    }
}
public class Application {
    public static void main(String[] args) {
      Teacher tea = new Teacher();
      tea.setName("tea");
    }
}

重写

重写：需要有继承关系，子类重写父类的方法！

​ 1. 方法名必须相同

​ 2.参数列表必须相同

​ 3.修饰符：范围可以扩大，但不能缩小 。public > protected > 默认 > private

​ 4.抛出的异常：范围可以缩小，但不能扩大。

  • 静态方法调用对象只和左边有关
class A extends B{
    public static void test(){  // static
        System.out.println("A");
    }
}

class B{
    public static void test(){  // static 
        System.out.println("B");
    }
}

public class Application {
    public static void main(String[] args) {
        // 方法的调用只和左边定义的数据类型有关
        A a = new A();
        a.test(); // 输出 A

        // 父类的引用指向了子类
        B b = new A();
        b.test(); // 输出 B
    }
}
  • 非静态调用方法构成重写，重写与静态方法无关
class A extends B{
    // Override 重写
    @Override  // 注解：有功能的注释
    public void test() {
        System.out.println("A");
    }
}

class B{
    public  void test(){
        System.out.println("B");
    }
}

public class Application {
    // 于上面静态方法有区别
    public static void main(String[] args) {
        A a = new A();
        a.test(); // 输出 A

        // 父类的引用指向了子类
        B b = new A(); // A子类重写了B父类的方法
        b.test(); // 输出 A
    }
}

多态

  • 即同一方法可以根据发送对象的不同而采用多种不同的行为方式。

  • 一个对象的实际类型的确定的，但可以指向对象的引用的类型有很多。

  • 多态存在的条件

  • 一个方法，如果子类没有重写父类的方法，则是调用父类的，否则调用子类的方法。

  • 多态注意事项：

    1. 多态是方法的多态，属性没有多态。
    2. 父类和子类，有联系
    3. 存在的关系：继承关系，方法需要重写，父类的引用指向子类对象。
class Person{
    public void run(){
        System.out.println("Person run");
    }
}
class Student extends Person{
    @Override
    public void run() {
        System.out.println("Student run");
    }
    public void eat(){
        System.out.println("eat");
    }
}
public class Application {
    public static void main(String[] args) {

        // 一个对象的实际类型是确定的
        // new Person();
        // new Student();

        // 可以指向的引用类型就不确定了：父类的引用指向子类

        // Student 能调用的方法都是直接的或者父类的
        Student s1 = new Student();
        // Person 父类能指向子类，但不能调用子类独有的方法
        Person s2 = new Student();
        Object s3 = new Student(); // Object 是所有类的父类

        // 能执行哪些方法主要看对象左边的类型
        s2.run(); // 输出Student run 子类重写父类的方法，执行子类的方法
        s1.run();

        ((Student) s2).eat(); // 强制类型转换
    }
}

instanceof

双目运算符 ， 左边是对象，右边可以是类或者对象;

class Person{
    public void run(){
        System.out.println("run");
    }
}
class Student extends Person{
    public void go(){
        System.out.println("go");
    }
}
class Teacher extends Person{

}
public class Application {
    public static void main(String[] args) {
        // Object > Person > Student
        // Object > Person > Teacher
        // Object > String
        // System.out.println(X instanceof  Y); 如果存在父子关系则编译通过，否则报错
        // X >= Y 则为 true
        Object object = new Student();
        System.out.println(object instanceof Student); // true
        System.out.println(object instanceof Person); // true
        System.out.println(object instanceof Object); // true
        System.out.println(object instanceof Teacher); // false
        System.out.println(object instanceof String);  // false

        Person person = new Student();
        System.out.println(person instanceof Student); // true
        System.out.println(person instanceof Person); // true
        System.out.println(person instanceof Object); // true
        System.out.println(person instanceof Teacher); // false
        // System.out.println(person instanceof String);  // 左右没有关系就报错

        Student stu = new Student();
        System.out.println(stu instanceof Student); // true
        System.out.println(stu instanceof Person); // true
        System.out.println(stu instanceof Object); // true
      // System.out.println(stu instanceof Teacher); // 没有关系就报错
      //  System.out.println(stu instanceof String);  // 没有关系就报错
    }
}

接口

  • 普通类：只有具体实现

  • 抽象类：具体实现和规范（抽象方法）都有！

  • 接口：只有规范！直接无法写方法专业的约束！约束和实现分离：面向接口编程

  • 接口就是规范，定义的是一组规则，体现了现实世界中“如果你是…则必须能…”的思想。如果你是天使，则必须能飞。如果你是汽车，则必须能跑。如果你好人，则必须干掉坏人;如果你是坏人，则必须欺负好人。

  • 接口的本质是契约，就像我们人间的法律一样。制定好后大家都遵守。

  • OO的精髓，是对对象的抽象，最能体现这一点的就是接口。为什么我们讨论设计模式都只针对具备了抽象能力的语言（比如c++、java、c#等)，就是因为设计模式所研究的，实际上就是如何合理的去抽象。

  • 声明类的关键字class , 声明接口的关键字interface

  • 接口的声明

    package main;
// interface 定义接口 ， 接口都需要有实现类
public interface Interface {
    // 接口中所有定义都是抽象类的 public abstract
    void add(String name);
    void del(String name);
    void update(String name);
    void query(String name);
}

  • 接口的实现类

    package main;
// 接口的实现类 ， 接口名 + Impl  , implements 实现的意思
// 实现了接口的类就需要重写接口的方法
//                                    多继承~利用接口
public class InterfaceImpl implements Interface , Use{ // 可以实现多个接口
    @Override
    public void add(String name) {

    }

    @Override
    public void ak() {

    }

    @Override
    public void del(String name) {

    }

    @Override
    public void update(String name) {

    }

    @Override
    public void query(String name) {

    }
}

  • 接口作用

    1. 约束
    2. 定义一些方法让多个人实现
    3. 接口方法默认用public abstract
    4. 接口属性默认用public static final
    5. 接口不能被实例化，接口中没有构造方法
    6. implements可以实现多个接口
    7. 必须重写接口里的每个方法
