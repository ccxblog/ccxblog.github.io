异常

  • 什么是异常

    • 在实际工作中，遇到的事情不可能是完美的。比如：你写某个模块，用户输入不一定合法，你的程序打开某个文件，这个文件可能不存在或者文件格式不对，你要读取数据库的数据，数据可能是空的等。
    • 异常的英文 Exception , 意思是例外。遇到这些例外或者说是异常，怎么让程序做出合理的处理。而不至于让程序奔溃。
    • 异常在程序执行中，影响了程序的正常执行顺序。

  • 简单分类

    • 检查性异常：最具代表性的是用户错误或者问题引起的异常，这是程序无法预见的。例如要打开一个不存在的文件时候，一个异常就产生了，这些异常在编译时不能被简单的忽略。

    • 运行时异常：是可以被程序员避免的异常，与检查性异常相反，运行时异常可以在编译的时候被忽略。

    • 错误：错误不是异常，而是脱离程序员控制的问题。错误在代码中通常被忽略，列如当栈溢出时错误就发生了，它在编译的时候是检查不到的。

  • 异常体现结构

    • Java把异常当作对象来处理，并定义一个基类java.lang.Throwable作为所有异常的超类。

    • 在Java API 中定义了许多异常类，这些异常类分为两大类 ： 错误（ERROR） 和 异常(Exception)

      • ERROR和Exception 的区别 ： ERROR 通常的灾难性的致命的错误，是程序无法控制和处理的，当出现这些错误的时候，JVM 通常会终止线程。而Exception 通常情况是可以被程序处理的，并且在程序中应该尽可能的处理这些异常。

  • 异常处理机制

    • 抛出异常

    • 抓取异常

    • 异常处理5个关键字 : try , catch , finally , throw , throws

    • 异常捕获快捷键 ctrl + alt + t 选择相应模块自动生成。

    public class Application {
    // 假设这个方法处理不了这个异常 ， 向上抛出异常
    public void test(int a , int b) throws ArithmeticException{
        if(b == 0){
            throw new ArithmeticException(); // 制造一个异常
        }
    }

    public static void main(String[] args) {

        int a = 1;
        int b = 0;
        try {
            // 判断异常 ， 可以主动抛出异常 throw
            if(b == 0) throw new ArithmeticException();
            System.out.println(a / b);
        }finally {

        }
        try { // try 监控区域
            System.out.println(a / b);
        }catch(Error e){ // catch （想要捕获的异常 ）
            System.out.println("捕获成功" );
        }catch (Exception ee){ // 可以捕获多个异常 ， 最大的异常要到最后面
            System.out.println("捕获成功" );
        }
        finally{ // 善后工作  ， 可有可无，不管有没有异常都会执行
            System.out.println("finally");
        }

        try {
            new Application().test(a, b);
        } catch (ArithmeticException e) {
            e.printStackTrace();
        }

    }
}

  • 自定义异常

    • 使用java内置的异常类可以描述在编程时出现的大部分异常情况，除此之外，用户还可以自定义异常。用户自定义异常类只需要继承Exception 类即可。

    • 自定义异常大体步骤

      1. 创建自定义异常类
      2. 在方法中通过throw 抛出异常对象。
      3. 如果在当前抛出异常的方法中处理异常，可以使用try - catch语句捕获并处理；否则在方法的声明处通过throw关键字指明要抛出给方法调用者的异常，继续进行下一步操作。
      4. 在出现异常方法的调用者中捕获并处理异常。
    package main;
// 自定义异常类
public class MyException extends Exception {
    // 传递数字
    private int a;
    public MyException(int a){
        this.a = a;
    }

    // toString: 异常的打印信息
    @Override
    public String toString() {
        return "MyException{" + "a=" + a + '}';
    }
}

// 测试方法
package main;
public class Application {

     static void test(int a) throws MyException{
        if(a > 10){
            throw new MyException(a); // 抛出
        }
    }
    public static void main(String[] args) {

         try{
             test(100);
         }catch (MyException e){
             System.out.println(e); // 打印：MyException{a=100}
         }
    }
}

集合

  • Java的集合存放于java.util 的包里，这是一个存放对象的容器。

    1. 集合只能存放对象，比如存储一个int数据 1 放入集合中，其实它会自动转换成Integer类后存入的，java中每一条基本类型都要对应的引用类型。
    2. 集合存放的是对多个对象的引用，对象本身还是存放在堆中。
    3. 集合可以存放不同类型，不限数量的数据。

  • Java集合可分为Set, Map ,List三大体系。

    Set：无序，不可重复的集合。

    List：有序，可重复的集合。

    Map：具有映射关系的集合。

HashSet

HashSet是 Set 的 典型实现 ， 大多数使用 Set 集合都用这个实现类 ， 平时说的 Set 说的就是 HashSet。HashSet按 Hash 算法来存储集合里面的元素，因此具有很好的存取和查找性能。

HashSet 的特点：

  • 不保证元素的排列顺序。

  • 不可重复

  • HashSet 不是线程安全的

  • 集合元素可以为NULL

    当向 HashSet 插入一个对象时 ， HashSet 会调用该对象的 HashCode() 来得到该对象的 HashCode 值 ， 后根据该值决定改对象的存储位置。

    如果两个元素的equals 方法返回 true ， 但它们的 hashcode（）值不相等，hashSet将会把它们存储在不同的位置。

package main;
import java.util.Set;
import java.util.HashSet;
public class Main {
    public static void main(String[] args) {
        Set set = new HashSet();
        set.add(1); // 添加元素
        set.add(2);
        set.add("a");
        set.add(null);
        set.add("abc");
		set.size();  // 长度
        set.remove(2); // 删除元素

        System.out.println(set.contains(1)); // 判断元素是否在集合里面
        System.out.println(set); // 输出整个集合
        for(Object obj : set) System.out.println(obj); // 遍历
		set.clear();  // 清空集合
        Set<String> S = new HashSet<String>();

        S.add("abc");
        // S.add(1); 报错 ， 只能存String

    }
}

TreeSet

TreeSet 可以确保元素处于排序状态。

TreeSet 支持两种排序方法：自然排序和定值排序， 默认情况是自然排序。

  • 自然排序：TreeSet会调用集合元素的compare To(Object obj) 方法来比较元素之间的大小关系，然后将集合元素升序排列。
package main;
import java.util.Set;
import java.util.TreeSet;

public class Main {
    public static void main(String[] args) {
        Set<Integer> set = new TreeSet<Integer>();

        set.add(4); set.add(3);set.add(1);
        set.remove(4);
        System.out.println(set);
        set.clear();
        
    }
}

  • 定值排序

    需要实现一个Comparator 接口的实现类对象 

package main;
import java.util.Set;
import java.util.TreeSet;

class Person implements Comparable<Person>{
    String name;
    int age;
    public Person(){

    }
    public Person(String s , int b){
        name = s ; age = b;
    }
    @Override
    public int compareTo(Person o) {
        if(this.age < o.age) return 1;
        else if(this.age > o.age) return  -1;
        else return 0;
    }

}

public class Main {
    public static void main(String[] args) {
        Set<Person> set = new TreeSet<Person>();

        Person a = new Person("abad" , 12);
        Person b = new Person("bcad" , 14);
        Person c = new Person("cdea" , 16);
        Person d = new Person("eadc" , 15);
        set.add(a);set.add(b);set.add(c);set.add(d);

        for(Person p : set) System.out.println(p.name + "--- " + p.age);
        // 年龄 输出 16 15 14 12
    }
}

List 与 ArrayList

  • List 代表一个元素有序，且重复的集合，集合中的每个元素都有其对应的顺序索引。

  • List允许重复元素，可以通过索引访问。

  • List默认按元素的添加顺序设置元素的索引。

  • List集合里添加了一些根据索引来操作集合的方法。

ArrayList 是 List 接口的金典实现

package main;
import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<String> list = new ArrayList<String>();

        list.add("a"); // 下标 0
        list.add("c");
        list.add("e");
        list.add("c"); // 下标 3 元素可以重复
        System.out.println(list);

        list.set(2 , "f"); // 修改
        System.out.println(list.indexOf("a")); // 查找第一次出现的下标
        System.out.println(list.lastIndexOf("c")); // 查找最后一次出现的下标
        System.out.println(list.get(2)); // 获取下标为 2 的元素 e
        list.add(1 , "f"); // 插入元素
        List<String> LL = list.subList(1 , 2); // 截取 【 ) 区间
        list.addAll(1 , LL); // 集合插入

    }
}

Map与HashMap

Map 是用于保存具有映射关系的数据 ， 因此Map 里有两组数据一组是 Key ， 一组是 Value 。 Key 和 Vaule 都可以是任何 引用类型 的数据 ， Map 中的 Key 不允许重复。 

package main;

import java.util.HashMap;
import java.util.Map;
import java.util.Set;

public class Main {
    public static void main(String[] args) {
        Map<String , Integer> map = new HashMap<String , Integer>();
        map.put("a" , 1); map.put("b" , 2); map.put("c" , 3); // 添加
        map.size(); map.remove("a"); // 根据 Key 删除
        System.out.println(map.get("a")); // 根据 Key 获取值
        map.containsKey("a"); // 是否存在对应 Key
        map.containsValue(1); // 是否存在 对应 Value

        map.keySet(); // 获取所有的 Key 值
        map.values(); // 获取所有的 value 值

        // 遍历
        Set<String> a = map.keySet();
        for(String s : a) System.out.println(map.get(a));
        
    }
}

TreeMap 有序的 Map 使用方法差不多。
