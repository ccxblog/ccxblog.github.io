## 运算符

java运算符有

  • 算术运算符：+ ， - ， * ， / ， % ， ++ ， –
  • 赋值运算符：=
  • 关系运算符：> , >= , < , <= , == , != ,instanceof
  • 逻辑运算符：&& , || , !
  • 位运算： & ， | ， ~ ， >> , <<
  • 条件运算符: ? :

Java的包机制

  • 为了更好的组织类，Java 提供了包机制， 用于区别类名的命名空间。

  • 包语句的语法格式：package pkg1 [.pkg2[.pkg3...]] ;且package 必须放在Java程序的最上面。

  • 一般公司利用公司域名倒置作为包名: com.ccxblog.www

  • 为了能够使用一个包的成员,我们需要再Java程序中明确导入该包 , 使用import 语句即可完成。

    import package1[. package2...].(classname|*);

JavaDoc

  • JavaDoc命令是用于生成自己的API文档的，它可以把我们的注释信息生成一份帮助文档，类似于Java的API文档。

    /**
 * @author 作者名
 * @version 版本号
 * @since 指明需要最早时期的jdk版本
 * @param 参数名
 * @return 返回值
 * @throws 异常抛出情况
 */
public class Main {
    String name;
    /**
     * @param name
     * @return
     * @throws Exception
     */
    public String test(String name) throws Exception{
        return name;
    }
}

总而言之，加在类上面就是类的注释，加在方法的前面就是方法的注释。

  • IDEA 生成JavaDoc

    通过IDEA生成Javadoc： Tools –> Generate JavaDoc –>
    注意要配置编码，如果不配送为生成乱码，还需要配置Output directory（输出文件存放目录）

注意编码配置-encoding UTF-8 -charset UTF-8

其中第一个参数 -encoding UTF-8 表示你的源代码（含有符合 JavaDoc 标准的注释）是基于 UTF-8 编码的，以免处理过程中出现中文等非英语字符乱码；第二个参数 -charset UTF-8 表示在处理并生成 JavaDoc 超文本时使用的字符集也是以 UTF-8 为编码，目前所有浏览器都支持 UTF-8，这样最具有通用性，支持中文非常好。
