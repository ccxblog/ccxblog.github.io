运算符

java运算符有

  • 算术运算符：+ ， - ， * ， / ， % ， ++ ， –

  • 赋值运算符：=

  • 关系运算符：> , >= , < , <= , == , != ,instanceof

  • 逻辑运算符：&& , || , !

  • 位运算： & ， | ， ~ ， >> , <<

  • 条件运算符: ? :

Scanner对象

  • java.util.Scanner是 Java5 的新特征，可以通过Scanner类来获取用户的输入。

  • 基本语法 Scanner cin = new Scanner(System.in);

  • 通过Scanner类的 next() 与 nextLine() 方法获取输入的字符串，在读取前我们一般需要使用hasNext() 于 hasNextLine() 判断是否还有输入数据。

    • next()
      1. 一定要读取到有效字符后才可以结束输入。
      2. 对有效字符之前遇到的空白, next() 方法会自动将其去掉。
      3. 只有输入有效字符后才将其后面的空白作为分隔符或结束符。
      4. next() 不能 得到带空格的字符串。

  • nextLine()

    1. 以 Enter 为结束符， 也就是说nextLine() 方法返回的是输入回车前的所有字符串。
    2. 可以获得空白。
    import java.util.Scanner; // 导入Scanner 包
public class Main {
    public static void main(String[] args) {
        // 创建一个扫描器对象 ， 用于接收键盘数据
        Scanner cin = new Scanner(System.in);

        // 如果还有下一个输入
        if(cin.hasNext()){
            // 使用next方法接收
            String s = cin.next();
            // 输出输入的字符串
            System.out.println(s);
        }
        // 使用完记得关闭掉，IO 流的类不关闭会占用一些资源。
        cin.close();

    }
}

  • Scanner 还有很多方法，列如用户可以调用nextByte(),nextDouble(),nextFloat,nextInt(),nextLine(),nextLong(),nextShot()

    等来输入各种类型的数据。同理还有相对应的has方法来判断是否还有对应类型的数据。

Java 流程控制

既然 Java 刚开始被称为C++ -- 所以它与C ++ 有许多相同的地方。C ++ 的 `if` , `switch` , `while`, `Do while`, `for` ,`break` , `continue` 等流程控制语句在 Java 里也同样适用 。Java 的`foreach` 语句( C++11 也有相关语法)为遍历集合 和 数组提供了很大的方便。


public class Main {
    public static void main(String[] args) {

        int[] a = {1 , 2 ,3 , 4 ,5 , 6};
        for(int x : a) System.out.print(x);

    }
}

Java的包机制

  • 为了更好的组织类，Java 提供了包机制， 用于区别类名的命名空间。

  • 包语句的语法格式：package pkg1 [.pkg2[.pkg3...]] ;且package 必须放在Java程序的最上面。

  • 一般公司利用公司域名倒置作为包名: com.ccxblog.www

  • 为了能够使用一个包的成员,我们需要再Java程序中明确导入该包 , 使用import 语句即可完成。

    import package1[. package2...].(classname|*);

JavaDoc

  • JavaDoc命令是用于生成自己的API文档的，它可以把我们的注释信息生成一份帮助文档，类似于Java的API文档。

    /**
 * @author 作者名
 * @version 版本号
 * @since 指明需要最早时期的jdk版本
 * @param 参数名
 * @return 返回值
 * @throws 异常抛出情况
 */
public class Main {
    String name;
    /**
     * @param name
     * @return
     * @throws Exception
     */
    public String test(String name) throws Exception{
        return name;
    }
}

总而言之，加在类上面就是类的注释，加在方法的前面就是方法的注释。

  • IDEA 生成JavaDoc

    通过IDEA生成Javadoc： Tools –> Generate JavaDoc –>
    注意要配置编码，如果不配送为生成乱码，还需要配置Output directory（输出文件存放目录）

注意编码配置-encoding UTF-8 -charset UTF-8

其中第一个参数 -encoding UTF-8 表示你的源代码（含有符合 JavaDoc 标准的注释）是基于 UTF-8 编码的，以免处理过程中出现中文等非英语字符乱码；第二个参数 -charset UTF-8 表示在处理并生成 JavaDoc 超文本时使用的字符集也是以 UTF-8 为编码，目前所有浏览器都支持 UTF-8，这样最具有通用性，支持中文非常好。
