Java的特性与优势

简单性

Java可以说是c/c++的纯净版本，它没有头文件，没有指针，也不用太在意内存管理，学习容易。

面向对象

面向对象将重点放在对象以及对象的接口上，面向对象更符合人的思维。

可移植性

Java的代码可以跨平台的移植，（write once , ran anywhere）写一次可以到处运行。就是说在Windows上写的程序可以在Linux上运行。

高性能

Java里面有个东西叫及时编译，它可以使某些代码的效率接近C/C++的效率。

分布式

Java可以写一些分布式的程序，Java可以说是为网络分布式环境所设计的。

动态性

Java本身是不具有动态性的，但是Java有个机制叫 反射 机制 ， 通过反射机制让Java具有的动态性。

多线程

Java支持多线程

安全性

Java适合用于网络的分布式环境，为了符合这个目标Java在安全性方面投入了很大的精力，使Java可以构建一些防病毒，防篡改的系统。

健壮性

Java吸收了C/C++ 的优点，去掉了C/C++影响中影响程序健壮性的部分；例如指针，内存管理，Java几乎不会造成计算机的崩溃，这是因为Java在每次运行的时候都会对内存进行一些检查。哪怕出现了一些问题也不会奔溃，因为Java有个异常机制，它可以通过捕获异常来解决所有的异常。

Java三大版本

JavaSE

这是Java的标准版，是Java的标准与核心。主要用于桌面程序和控制台程序的开发。

JavaME

JavaME相当于java的微缩版，主要用于嵌入式的开发(手机程序，小家电)。目前差不多已经死掉了。

JavaEE

JavaEE是java的企业版，主要用于Web端，服务器端的开发。

JDK , JRE , JVM

JDK

JDK ——Java开发者工具。

它除了包含 JRE ，JVM之外还包含一些自己用于程序开发的工具。

JRE

JRE ——Java运行时环境

JRE包含JVM及一些简单的函数库等

JVM

Java 虚拟机，可以当成一种规范，JAM可以用软件来实现，也可以用硬件来实现。它就相当于在所有的操作系统上模拟了一个小CPU去处理Java相关的东西。Java跨平台的核心就是JVM。

Java的运行机制

编译型与解释型语言

编译型语言：把做好的源程序全部编译成二进制代码的可运行程序。然后，可直接运行这个程序。
解释型语言：把做好的源程序翻译一句，然后执行一句，直至结束！
区别：
编译型语言，执行速度快、效率高；依靠编译器、跨平台性差些。
解释型语言，执行速度慢、效率低；依靠解释器、跨平台性好。

JAVA–编译型

​ Java可以说是编译型的。Javac 程序是一个Java的编辑器。它将 .java 文件编译成 .class 文件 并发送到 Java 虚拟机。

JAVA–解释型

Java代码经过编译后还是不能直接运行，它是解释运行在 JVM 上的 ， 所以Java也是解释型的
