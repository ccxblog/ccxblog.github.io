SpringMVC 的 hello world

  1. 创建一个 JavaEE 工程
  2. 导包
    • 核心容器模块
    • web模块
  3. 配置前端控制器：SpringMVC 思想是有一个前端控制器（DispatcherServlet）能拦截所有请求，并智能派发 ， 这个前端控制器是一个 Servlet。所以应该在 web.xml 中配置这个 Servlet。
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://xmlns.jcp.org/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd"
         version="4.0">

    <servlet>
        <servlet-name>dispatcherServlet</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>

        <init-param>
            <!-- contextConfigLocation 指定springMVC 配置文件位置 -->
            <param-name>contextConfigLocation</param-name>
            <!-- 表示当前类路径下的 bean.xml -->
            <param-value>classpath:bean.xml</param-value>
        </init-param>

        <!-- 有 load-on-startup 后就会在服务器启动的时候创建对象 ， 值越小优先级越高  -->
        <load-on-startup>1</load-on-startup>
    </servlet>

    <servlet-mapping>
        <servlet-name>dispatcherServlet</servlet-name>
        <!-- 拦截所有请求 -->
        <!-- / 和 /* 都是拦截所有请求 -->
        <!-- /* 会拦截jsp页面导致无法显示 -->
        <!-- Tomcat 中有个默认的defaultServlet配置的也是 / 用来处理静态资源除了servlet外都是静态资源-->
        <!-- 这相当于前端控制器的 / 禁用了 Tomcat 服务器的 defaultServlet -->
        <url-pattern>/</url-pattern>
    </servlet-mapping>


</web-app>
  1. 在 Spring 里面配置扫描所有组件
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
                           http://www.springframework.org/schema/context http://www.springframework.org/schema/context/spring-context.xsd">

    <context:component-scan base-package="com.xixi.controller"></context:component-scan>

    <!-- 配置视图解析器 能帮我们拼接页面地址-->
    <bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
        <property name="prefix" value="/WEB-INF/pages/"></property>
        <property name="suffix" value=".jsp"></property>
    </bean>

</beans>
  1. IDEA 要在项目发布中，添加 lib 依赖 (坑老子) 重点不然 404

  1. 写一个Controller类 , 添加@Controller 注解 , 下面的方法添加 @RequestMapping（请求映射）
package com.xixi.controller;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
public class MyfirstController {
    // / 代表当前项目下开始 ， 处理当前项目下的 hello 请求， /可以不写
    @RequestMapping("/hello")
    public String myFirstRequest(){
        System.out.println("okkkkk");
        // 返回跳转页面 , 配置的视图解析器会根据前缀后缀自动帮我们拼接字符串
        // 是一次请求转发操作
        return "success";
    }
}
  • 运行流程
  1. 客户端发送 hello 即 http://localhost:8080/hello
  2. 来到 Tomcat 服务器
  3. SpringMVC的前端控制器接收到请求
  4. 接着对 请求地址 与 ReqeustMapping 的那个匹配
  5. 利用反射调用匹配类的指定方法
  6. 方法执行完后有一个返回值，SpringMVC认为这是返回的页面地址
  7. 视图解析器拼接字符串请求转发到完整页面。

RequestMapping 常用两点

  1. 可以在路径上写通配符 : 
@RequestMapping("/a/*/?bc")
public String get1(){
    return "success";
}
  1. 可以加 { 变量名 } 占位符配合 @PathVariable注解 获取路径上的占位符的值，占位符只能占一层路径
@RequestMapping("/user/{id}")
public String get2(@PathVariable("id") String id){
    System.out.println(id);
    return "success";
}

REST风格

REST:(资源)表现层状态转化

系统希望以非常简洁的 Url 地址来请求

  • Jew阶段对资源进行增删改查：

    • /getBook?id=1 : 查询图书
    • /deleteBook?id=1 : 删除图书
    • /updateBook?id=1 : 更新图书
    • /addBook ： 添加图书

  • REST 推荐使用 /资源名/资源标识符

    • /book/1 : GET请求 – 查询 1 号图书
    • /book/1 : POST请求 – 添加 1 号图书
    • /book/1 : DELETE请求 – 添加 1 号图书
    • /book/1 : PUT请求 –更新 1 号图书

从页面上只能发送 GET 和 POST 请求 ， 但 SpringMVC 中有个 Filter , 它可以把普通的请求转为规定形式的请求

  • 首先配置这个 Filter
<filter>
    <filter-name>HiddenHttpMethodFilter</filter-name>
    <filter-class>org.springframework.web.filter.HiddenHttpMethodFilter</filter-class>
</filter>

<filter-mapping>
    <filter-name>HiddenHttpMethodFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
  • 在前端中 ，创建一个 post 类型的表单 ， 这个表单带一个 _method参数 ，这个参数就是我们的请求
<form action="book/1" method="post">
    <input type="hidden" name="_method" value="put">
    <input type="submit" value="put" />
</form><br/>

SpringMVC 获取请求带来的各种信息

  1. 前端有一个链接
<a href="book?username=abc">查询图书</a>

后端可以取一个和 username 一样的变量接收

@RequestMapping("/book")
public String book(String username){
    System.out.println(username);
    return "success";
}
  1. 使用@RequestParam 注解可以起别名 ， required = false 表示参数不必须，请求没这个参数也没事,默认是true，不带会出错。 defaultValue 设置默认值 ， 默认为 null
@RequestMapping("/book")
public String book(@RequestParam(value = "username" , required = false ,defaultValue = "aa") String name){
    System.out.println(name);
    return "success";
}
  1. 还可以接收原生 API
@RequestMapping("/book/user")
public String add(HttpServletRequest request ){

    return "success";
}

其它两个参数，不具体研究了

  1. @CookieValue 获取 Cookie 的值

  2. @RequestHeader 获取请求头的值

  • 还可以直接获取类信息

例如现在有个 Book 类 ， Book 类 里面有一个 Person 类属性 , 并添加对应方法

package com.xixi.controller;

import org.springframework.context.annotation.Bean;

public class Book {

    private Integer id;
    private Integer name;
    private Person person;

    public Book() {
    }

    public Book(Integer id, Integer name, Person person) {
        this.id = id;
        this.name = name;
        this.person = person;
    }

    public Integer getId() {
        return id;
    }

    public void setId(Integer id) {
        this.id = id;
    }

    public Integer getName() {
        return name;
    }

    public void setName(Integer name) {
        this.name = name;
    }

    public Person getPerson() {
        return person;
    }

    public void setPerson(Person person) {
        this.person = person;
    }

    @Override
    public String toString() {
        return "Book{" +
                "id=" + id +
                ", name=" + name +
                ", person=" + person +
                '}';
    }
}

    <input type="submit" value="提交">
</form>

我们可以在 Controller 层直接写 Book 接收这个表单

@RequestMapping("/book")
public String book(Book book){
    System.out.println(book);
    return "success";
}

使用SpringMVC 自带编码 Filter 解决乱码问题

只要在 web.xml 配置就行

<filter>
    <filter-name>filter</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <!-- 指定解决 post 请求乱码 -->
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
    <!-- 顺手 解决响应编码 -->
    <init-param>
        <param-name>forceEncoding</param-name>
        <param-value>true</param-value>
    </init-param>
</filter>
<filter-mapping>
    <filter-name>filter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>

SpringMVC 的数据输出

  1. 使用原生 API
  2. 在方法处传入 Map , Model 或 ModelMap 实现将数据保存在 request域中 
@RequestMapping("/hello1")
public String hello(Map<String , Object> map){
    map.put("name" , "你好");
    return "success";
}

它们3的关系：Map ， Model ， ModelMap 最终的实现都是 SpringMVC 中的 BindingAwareModelMap ,

BindingAwareModelMap 继承了 ExtendedModelMap 而 ExtendedModelMap 继承了 ModelMap 并实现了Model ， 而ModelMap 是继承 LinkedHashMap<String, Object> 好绕。。。。

  1. 方法的返回值可以是 ModeAndView 类型
@RequestMapping("/hello1")
public ModelAndView hello(Map<String , Object> map){
    // 生成对象时候传入视图名字 ： 视图解析器会拼串
    ModelAndView mv = new ModelAndView("success");
    mv.addObject("name" , "你好");
    return mv;
}

转发与重定向

Spring 如果返回字符串默认是 转发 ， 并且会经过视图解析器进行拼串 ， 我们可以加上 forward:转发路径 前缀实现请求 转发，使用 forward:前缀的化就不会经过视图解析器的拼串。

@RequestMapping("/hello1")
public String hello(Map<String , Object> map){
    return "forward:/hello.jsp";
}

如果是重定向的话可以 使用 redirect:重定向路径 , 原始的 重定向需要写项目路径 ， 使用 redirect 前缀可以不加项目路径， SpringMVC 会自动帮我们拼接

@RequestMapping("/hello1")
public String hello(Map<String , Object> map){
    return "redirect:/hello.jsp";
}

Spring MVC - ajax

首先要导入一些包。然后方法返回 JSON 数据可以加上 @ResponseBody 注解，如果返回对象自动将对象转为 JSON格式

@ResponseBody
@RequestMapping("/hello1")
public Book hello(Map<String , Object> map){
    Book book = new Book();
    return book;
}

与之对应的是 @RequestBody 可以接收 JSON 数据 直接封装成对象

@ResponseBody
@RequestMapping("/hello1")
public Book hello(@RequestBody Book book){
    System.out.println(book);
    return book;
}

拦截器

SpringMVC 提供了拦截器机制 ， 允许运行目标方法前进行拦截工作 ， 或者目标方法后进行一些其它处理 ， 它是一个接口

package org.springframework.web.servlet;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import org.springframework.lang.Nullable;

public interface HandlerInterceptor {
    // 目标方法运行前执行 , true 放行 , false 不放行
    default boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        return true;
    }
	// 目标方法调用后	
    default void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, @Nullable ModelAndView modelAndView) throws Exception {
    }
	// 请求完成后；即来到目标页面后
    default void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, @Nullable Exception ex) throws Exception {
    }
}
  • 创建一个拦截器
package com.xixi.controller;

import org.springframework.web.servlet.HandlerInterceptor;
import org.springframework.web.servlet.ModelAndView;
public class MyHandlerInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(javax.servlet.http.HttpServletRequest request, javax.servlet.http.HttpServletResponse response, Object handler) throws Exception {
        return false;
    }

    @Override
    public void postHandle(javax.servlet.http.HttpServletRequest request, javax.servlet.http.HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {

    }

    @Override
    public void afterCompletion(javax.servlet.http.HttpServletRequest request, javax.servlet.http.HttpServletResponse response, Object handler, Exception ex) throws Exception {

    }
}
  • 在 SpringMVC 中注册 拦截器 ， 配置这个拦截器拦截那些请求的方法
    <!-- 拦截器 -->
    <mvc:interceptors>
        <!-- 默认拦截所有请求 -->
        <bean class="com.xixi.controller.MyHandlerInterceptor"> </bean> 

<!--        <mvc:interceptor>-->
<!--            &lt;!&ndash; 只拦截 /test 请求 &ndash;&gt;-->
<!--            <mvc:mapping path="/test"/>-->
<!--           <bean class="com.xixi.controller.MyHandlerInterceptor"> </bean>-->
<!--        </mvc:interceptor>-->

    </mvc:interceptors>
  • 如果有多个拦截器 ， 拦截器按照配置顺序的 preHandle 是顺序执行 ， postHandle 和 afterCompletion 是逆序执行

运行流程图解

  1. 用户向服务器发送请求，请求被SpringMVC 前端控制器 DispatcherServlet捕获；

  2. DispatcherServlet对请求URL进行解析，得到请求资源标识符（URI）:判断请求URI对应的映射

    • 不存在：

      • 再判断是否配置了mvc:default-servlet-handler：
      • 如果没配置，则控制台报映射查找不到，客户端展示404错误
      • 如果有配置，则执行目标资源（一般为静态资源，如：JSP,HTML）

    • 存在：

      • 执行下面流程

  3. 根据该URI，调用HandlerMapping获得该Handler配置的所有相关的对象（包括Handler对象以及Handler对象对应的拦截器），最后以HandlerExecutionChain对象的形式返回；

  4. DispatcherServlet 根据获得的Handler，选择一个合适的HandlerAdapter。

  5. 如果成功获得HandlerAdapter后，此时将开始执行拦截器的preHandler(…)方法【正向】

  6. 提取Request中的模型数据，填充Handler入参，开始执行Handler（Controller)方法，处理请求。在填充Handler的入参过程中，根据你的配置，Spring将帮你做一些额外的工作：

​ ① HttpMessageConveter： 将请求消息（如Json、xml等数据）转换成一个对象，将对象转换为指定的响应信息

​ ② 数据转换：对请求消息进行数据转换。如String转换成Integer、Double等

​ ③ 数据根式化：对请求消息进行数据格式化。 如将字符串转换成格式化数字或格式化日期等

​ ④ 数据验证： 验证数据的有效性（长度、格式等），验证结果存储到BindingResult或Error中

  1. Handler执行完成后，向DispatcherServlet 返回一个ModelAndView对象；

  2. 此时将开始执行拦截器的postHandle(…)方法【逆向】

  3. 根据返回的ModelAndView（此时会判断是否存在异常：如果存在异常，则执行HandlerExceptionResolver进行异常处理）选择一个适合的ViewResolver（必须是已经注册到Spring容器中的ViewResolver)返回给DispatcherServlet，根据Model和View，来渲染视图

  4. 在返回给客户端时需要执行拦截器的AfterCompletion方法【逆向】

  5. 将渲染结果返回给客户端
