Spring

  1. Spring 是一个轻量级的开源的 JavaEE 框架

  2. Spring 有两个核心部分 ：IOC 和 AOP

    • IOC：控制反转 ， 把创建对象过程交给 Spring 进行管理
    • AOP：面向切面，不修改源代码的情况下进行功能的增强

  3. Spring 特点

    • 方便解耦，简化开发
    • AOP编程的支持
    • 方便程序的测试
    • 方便整合其它框架
    • 方便进行事务的操作
    • 降低 ApI 的开发难度

IOC 容器

1.什么是 IOC

控制反转 ， 把对象创建和对象之间的调用过程，交给 Spring 进行管理 ， 目的是使耦合度降低。

2. IOC 底层原理

  1. xml 解析
  2. 工程模式
  3. 反射

  • IOC过程

    1. 首先要在 xml 中配置创建的对象

    <bean id = "user" class = "com.xixi.spring.User"> </bean>

    1. 可以创建一个工厂类
    class UserFactory{
    public static User getUser(){
        // 获取class属性值
        String classVlaue = class属性值  // xml 解析
       // 得到反射对象     
        Class clazz = Class.forName(classValue);
        // 通过反射创建对象
        return (User) clazz.newInstance();
    }
}
  • IOC 接口

IOC思想基于 IOC 容器完成 ， IOC 容器底层就是对象工程 ， Spring 提供 IOC 容器实现的两种方式：

  1. BeanFactory : IOC 容器基本的实现，是 Spring 内部的使用接口 ， 不提供开发人员进行使用 , 加载配置文件的时候不会创建对象，在获取对象的时候才去使用对象。
public class TestSpring5 {
    @Test
    public void test(){

        BeanFactory context = new ClassPathXmlApplicationContext("bean1.xml");
        
        // 这时候才创建对象
        User user = context.getBean("user", User.class);
        user.add();
    }
}
  1. ApplicationContext ： 是 BeanFactory 接口的子接口 ， 提供更多强大的功能，一般使用这个，加载配置文件的时候就会创建对象
public class TestSpring5 {
    @Test
    public void test(){

        // 获取的时候就创建对象了
        ApplicationContext context = new ClassPathXmlApplicationContext("bean1.xml");
        
        User user = context.getBean("user", User.class);
        user.add();
    }
}

虽然用 BeanFactory 更好一点，但是我们平时使用 Spring 框架基本都是进行 Web 开发，所以一些资源在服务器启动的时候加载会好一点，而不在操作中才完成，用空间换时间。

IOC 操作 bean 管理 (FactoryBean)

Spring 有两种 bean ， 一种普通 bean ， 另外一种工厂bean(FactoryBean)

普通 bean ： 在配置文件中定义 bean 类型就是返回值

工厂 bean ：在配置文件定义 bean 类型可以和返回类型不一样 ， 实现就是创建一个类，实现 FactoryBean 接口

  • Bean 的作用域

  1. 在 Spring 里面 ，可以设置创建 bean 实例是单例还是多例(默认是单例)

  2. 在 Spring配置文件的 bean 标签里面的 scope 属性设置单例还是多例

    • 设置单例 (默认)
    <bean id="user" class="com.xixi.spring5.User" scope="singleton"></bean>
    • 设置多例 ，在调用getBean()方法的时候创建多实例对象
    <bean id="user" class="com.xixi.spring5.User" scope="prototype"></bean>
  • Bean 的生命周期
  1. 通过无参构造器创建 bean实列
  2. 为 bean 的属性设置值和对其它 bean 引用 （调用set方法）
  3. 调用 bean 的初始化方法 （需要配置初始化方法）
  4. bean可以使用了 （获取到了对象）
  5. 容器关闭的时候，调用 bean 的销毁方法 （需要配置销毁方法）
  • 加上 bean 的后置处理器后 就会有 7 步
  1. 通过无参构造器创建 bean实列
  2. 为 bean 的属性设置值和对其它 bean 引用 （调用set方法）
  3. 在初始化之前执行后置处理器中的方法
  4. 调用 bean 的初始化方法 （需要配置初始化方法）
  5. 在初始化后执行后置处理器中的方法
  6. bean可以使用了 （获取到了对象）
  7. 容器关闭的时候，调用 bean 的销毁方法 （需要配置销毁方法）

具体配置步骤是，创建一个类实现 BeanPostProcessor

package com.xixi.spring5;

import org.springframework.beans.BeansException;
import org.springframework.beans.factory.config.BeanPostProcessor;

public class User implements BeanPostProcessor {
    
    // 初始化后执行的方法
    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        return null;
    }

    // 初始化前执行的方法
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        return null;
    }
}

AOP

什么是 AOP

AOP ： 面向切面编程 ， 指 不通过修改源代码方式添加新的功能 ， 可以对业务各个部分进行隔离 ， 从而使业务之间的耦合度降低 ， 提高可重用性。

AOP 底层原理

  1. AOP 底层使用动态代理方式 ， 有两种情况动态代理

    • 有接口的情况 ， 使用 JDK 动态代理 ， 创建接口实现类代理对象增强类中的方法。

    JDK 动态代理实现 ， 现在有一个 UserDaoImpl 类 实现了 UserDao 接口 ， 里面有个 add 方法 ， 我们要对add 方法进行扩展。

    package com.xixi.spring5.dao;

import org.springframework.stereotype.Repository;

@Repository
public class UserDaoimpl implements UserDao {
    @Override
    public void add() {
        System.out.println("UserDao add");
    }
}

    首先创建一个动态代理类 ， 在里面创建 UserDaoImpl 类的 接口数组class 对象 。 同时还要创建一个类实现 InvocationHandler 接口 ，并重写 invoke 方法 ， invoke 方法会在 代理类调用方法的时候执行。通过 Proxy.newProxyInstance（类加载器 , 代理对象实现的接口数组 ，InvocationHandler实现类）; 得到 UserDao 接口实现类的代理对象。

    package com.xixi.spring5.dao;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public  class JDKProxy {
    public static void main(String[] args) {
        Class [] interfaces = {UserDao.class};
        UserDao userDao = new UserDaoimpl();
        UserDao userDaoimpl = (UserDao) Proxy.newProxyInstance(JDKProxy.class.getClassLoader(), interfaces, new MyProxy(userDao));
        userDaoimpl.add();
    }
}

class  MyProxy implements InvocationHandler{

    private Object obj;

    public MyProxy(Object obj){
        this.obj = obj;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {

        System.out.println("方法增强执行之前....");
        Object o = method.invoke(obj, args);
        System.out.println("方法增强执行之后....");

        return o;
    }
}
  • 没接口的情况 ， 使用 CGLIB 动态代理 ， 创建子类的代理对象，增强类中的方法。

AOP术语

  1. 连接点 ： 类里那些方法可以被增强 ， 这些方法称为连接点。

  2. 切入点：实际被增强的方法

  3. 通知：实际增强的逻辑部分称为通知

    • 前置通知 ：方法执行前执行
    • 后置通知 ：方法执行后执行
    • 环绕通知 ：方法执行的前面后面都执行
    • 异常通知 ： 增强方法出现异常执行
    • 最终通知 ：类似 try-cath里的 finally 不管有没有异常都执行

  4. 切面 ： 把通知应用到切入点的过程就叫切面

AOP 操作

Spring 框架一般是基于 AspectJ 实现 AOP操作，AsppectJ 不是 Spring 的组成部分 。

  1. 引入相关依赖 (红色)

  1. 切入点表达式

    • 作用：知道对那个类的那个方法进行增强。
    • 语法结构 execution([权限修饰符] [返回类型] [类全路径] [方法名称] ([参数列表]))

    这里修饰符可以不写 ，返回类型可以用 * 匹配

对 com.xixi.java.Book类的 add 方法进行增强:

execution(* com.xixi.java.Book.add(..) )

对 com.xixi.java.Book类的 所有方法 进行增强:

execution(* com.xixi.java.Book.*(..) )

对 com.xixi.java包里的 所有 方法进行增强:

execution(* com.xixi.java.*.*(..) )

  1. 创建 User的代理类 UserProxy 并添加 @Component 注解 和 @Aspect注解表示要生成代理对象 , 可以在 UserProxy 配置相应的通知。
package com.xixi.spring5.aopanno;

import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.*;
import org.springframework.stereotype.Component;

@Component
@Aspect
public class UserProxy {

    // 前置通知
    @Before("execution(* com.xixi.spring5.aopanno.User.add(..) )")
    public void before(){
        System.out.println("前置通知");
    }
    // 最终通知 出现异常也会执行
    @After("execution(* com.xixi.spring5.aopanno.User.add(..) )")
    public void after(){
        System.out.println("最终通知 -- 方法执行之后执行");
    }

    // 后置通知 ， 出现异常不会执行
    @AfterReturning("execution(* com.xixi.spring5.aopanno.User.add(..) )")
    public void AfterReturning(){
        System.out.println("后置通知 -- 方法返回值之后执行");
    }

    // 环绕通知
    @Around("execution(* com.xixi.spring5.aopanno.User.add(..) )")
    public void around(ProceedingJoinPoint proceedingJoinPoint) throws Throwable {
        System.out.println("环绕通知前");
        proceedingJoinPoint.proceed();
        System.out.println("环绕通知后");
    }

    // 异常通知
    @AfterThrowing(value = "execution(* com.xixi.spring5.aopanno.User.add(..) )")
    public void afterThrowing(){
        System.out.println("异常通知");
    }

}

Spring 事务管理

Spring 事务管理有两种方式

  1. 编程式事务管理 ： 手动写 try- catch-finally 方法
  2. 声明式事务管理 : 有基于xml 配置方式 和 基于 注解方式 (常用)

Spring 事务管理 API :PlatformTransactionManager接口 ， 代表事务管理器 , 这个接口针对不同框架有不同的实现类 ， 常用的实现类是 DataSourceTransactionManager , 在 xml 对该类配置 后 ， 在需要进行事务管理的类 或 方法上面添加 @Transactional 注解即可 ， 出现异常会自动回滚。

事务的传播行为

Spring 框架事务的传播行为有 7 种 ， 常用的有两种 ，假如有两个 方法 add 和 update ， 现在我们在 add 中调用 update 方法 ；

  1. REQUIRED （默认）:
    • 如果 add 方法上面添加了事务 ， update上面没加事务 ， 则update使用 add方法里面的事务。
    • 如果 add 方法上面没事务 ， update 上面有事务 ， 则 add创建新的事务。
  2. REQUIRED_NEW :
    • 使用 add 调用 update 无论add是否有事务都创建一个新的事务。

配置方法是 ： @Transactional (propagation = “”)

  • 隔离级别

通过@Transactional (isolation = “”) 可以设置 4 个隔离级别

Spring5 整合 Log4j2

  1. 引入 jar 包

  1. 配置 xml 文件 , 名字固定为 log4j2.xml
<?xml version="1.0" encoding="UTF-8"?>
<!--日志级别以及优先级排序: OFF > FATAL > ERROR > WARN > INFO > DEBUG > TRACE > ALL -->
<!--Configuration后面的status用于设置log4j2自身内部的信息输出，可以不设置，当设置成trace时，可以看到log4j2内部各种详细输出-->
<configuration status="DEBUG">
    <!--先定义所有的appender-->
    <appenders>
        <!--输出日志信息到控制台-->
        <console name="Console" target="SYSTEM_OUT">
            <!--控制日志输出的格式-->
            <PatternLayout pattern="%d{yyyy-MM-dd HH:mm:ss.SSS} [%t] %-5level %logger{36} - %msg%n"/>
        </console>
    </appenders>
    <!--然后定义logger，只有定义了logger并引入的appender，appender才会生效-->
    <!--root：用于指定项目的根日志，如果没有单独指定Logger，则会使用root作为默认的日志输出-->
    <loggers>
        <root level="info">
            <appender-ref ref="Console"/>
        </root>
    </loggers>
</configuration>

Spring5 整合 JUnit4

  1. 导入 test jar包

  2. 新建一个类 添加如下两个注解

package com.xixi.spring5;

import org.junit.Test;
import org.junit.runner.RunWith;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.test.context.ContextConfiguration;
import org.springframework.test.context.junit4.SpringJUnit4ClassRunner;

@RunWith(SpringJUnit4ClassRunner.class) // 单元测试框架
@ContextConfiguration("classpath:bean.xml") // 加载配置文件
public class JUnit4Test {

    @Autowired
    private Book book;

    @Test
    public void test(){
        book.add();
    }
}
