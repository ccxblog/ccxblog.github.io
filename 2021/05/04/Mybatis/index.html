Mybatis 简介

•MyBatis 是支持定制化 SQL、存储过程以及高级映射的优秀的持久层框架。

•MyBatis 避免了几乎所有的 JDBC 代码和手动设置参数以及获取结果集。

•MyBatis可以使用简单的XML或注解用于配置和原始映射，将接口和Java的POJO（Plain Old Java Objects，普通的Java对象）映射成数据库中的记录.

第一个MyBatis

  1. 创建一个表

  2. 创建这个表的 JavaBean

  3. 创建 对这个表的 dao 接口

  4. 导入包

  5. 创建 映射sql 的xml ， 相当于对接口的实现

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">

<!--  namespace: 名称空间 ，写接口的全类名 , 告诉Mybatis这个配置文件是实现那个接口的 -->
<mapper namespace="com.xixi.mybatis.dao.EmployeeDao">

    <!-- id 是方法名字 , 配置方法的实现 -->
    <!-- resultType 返回类型 -->
    <!-- #{属性名} 取出传递过来的某个参数值 -->
    <select id="getEmployeeById" resultType="com.xixi.mybaits.pojo.Employee">
    select * from t_employee where id = #{id}
    </select>


</mapper>
  1. 配置 mybatis 全局配置文件
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration>
    <environments default="development">
        <environment id="development">
            <transactionManager type="JDBC"/>
            <!-- 配置连接池 -->
            <dataSource type="POOLED">
                <property name="driver" value="com.mysql.jdbc.Driver"/>
                <property name="url" value="jdbc:mysql://localhost:3306/mybaits"/>
                <property name="username" value="root"/>
                <property name="password" value="123456"/>
            </dataSource>

        </environment>
    </environments>

    <!-- 引入我们自己编写的每一个接口的实现文件 -->
    <mappers>
        <!--  表示从类路径下找资源-->
        <mapper resource="mybatis/EmployeeDao.xml"/>
    </mappers>

</configuration>
  1. 编写测试类测试
package com.xixi.test;

import com.xixi.mybaits.pojo.Employee;
import com.xixi.mybatis.dao.EmployeeDao;
import org.apache.ibatis.io.Resources;
import org.apache.ibatis.session.SqlSession;
import org.apache.ibatis.session.SqlSessionFactory;
import org.apache.ibatis.session.SqlSessionFactoryBuilder;
import org.junit.Test;

import java.io.IOException;
import java.io.InputStream;

public class MybatisTest {

    @Test
    public void test() throws IOException {
        // 根据全局配置文件创建一个 SqlSessionFactory
        // SqlSessionFactory : 负责创建 SqlSession 对象
        // SqlSession : 代表和数据库的一次会话
        String resource = "mybatis.xml";
        InputStream inputStream = Resources.getResourceAsStream(resource);
        SqlSessionFactory sessionFactory = new SqlSessionFactoryBuilder().build(inputStream);

        // 获取和数据库的一次会话 , getConnection()
        Employee employee = null;
        SqlSession openSession = null;
        try {
            openSession = sessionFactory.openSession();
            // 操作数据库 , 获取 dao 接口的实现
            EmployeeDao employeeDao = openSession.getMapper(EmployeeDao.class);
            // 执行方法获取结果
            employee = employeeDao.getEmployeeById(1);
        } catch (Exception e) {
            e.printStackTrace();
        }finally {
            if(openSession != null) openSession.close();
        }
        // 如果是 增删改 需要手动提交
       //  openSession.commit();
        System.out.println(employee);
    }
}

总结就是 ： 根据配置文件 得到一个输入流 通过配置文件的输入流 得到 SqlSessionFactory ， 然后开启会话 ，实例化对象 ， 并执行 ， 如果是 增删改 方法最后记得手动提交 , 可以在创建 openSession 的时候设置自动提交openSession = sessionFactory.openSession(true);

MyBatis 全局配置

先来个起始配置文件 ， MyBatis 配置的标签是有顺序的 ， 不能乱写 

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration
        PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">

<configuration>

</configuration>

  1. <properties > </properties> ： 引用外部配置文件

  2. settings : 这是 MyBatis 中极为重要的调整设置，它们会改变 MyBatis 的运行时行为。, 设置参数可以查看官方文档的表

  3. plugins : 插件 mybatis 中的一个强大功能

  4. environments : 配置多个具体的环境 ， 每个（environment）环境都需要配置一个事务管理器（transactionManager） 和 数据源（dataSource）。 后来 事务管理器和数据源由 Spring 来做

<environments default="development">

    <environment id="development">
        
        <!-- 事务管理器 -->
        <transactionManager type="JDBC"/>
        <!-- 配置连接池 -->
        <dataSource type="POOLED">
            <property name="driver" value="com.mysql.jdbc.Driver"/>
            <property name="url" value="jdbc:mysql://localhost:3306/mybaits"/>
            <property name="username" value="root"/>
            <property name="password" value="123456"/>
        </dataSource>
        
    </environment>

</environm
  1. databaseIdProvider：数据库移植 ，可以根据不同的数据库厂商执行不同的语句
  2. mappers : 写好的 sql 映射文件需要使用 mappers 注册
<mappers>
    <!--  表示从类路径下找资源-->
    <mapper resource="mybatis/EmployeeDao.xml"/>
    <!-- 磁盘路径 或 网络资源 -->
    <mapper url=""> </mapper>
    <!-- 引用接口的全类名 -->
    <!-- 使用class 需要把和接口同名的xml文件放在同一个包下-->
    <mapper class=""> </mapper>
</mappers>
  • 还可以在 mappers 中配置 包扫描 ，前题是 xml 和 接口文件必须是在同一个包下 ， 但为了分开 ， 我们可以把 xml 文件放在 resources 资源文件中 ，包路径 和 类路径的接口取一样的包名

<mappers>
    <package name="com.xixi.mybatis.dao"/>
</mappers>

这就叫约定大于配置吗…..

SQLMapper

  • mapper 中能写的标签
    1. cache : 缓存有关
    2. cache-ref ： 缓存有关
    3. resultMap : 自定义结果集封装规则
    4. sql : 抽取可以重用 sql
    5. delete , update , insert : 输出 , 更新 ，插入
<!-- 让mybatis 自动将自增 id 赋值给传入的Employee 对象 的id 属性
        useGeneratedKeys="true"
        keyProperty="id"
        最后可以调用传入对象的 getid 方法可以获取到id值 -->
<insert id="add" useGeneratedKeys="true" keyProperty="id">
    insert into t_employee(`name` , gender , email) value (#{name} , #{gender} , #{email})
</insert>
6. select : 查询
   - 传一个参数
     - 基本类型 取值：# {随便写}
   - 多个参数
     - #{参数名} 取值无效 ， 可用 0 ，1 参数索引代替 ，mybatis会把多个参数 封装在 map 中 ， 我们可以告诉 mybatis 传入参数的时候按照我们的规则来
   - 传入map
     - 自己封装下传 map
   - 传入pojo
     - 取值 ： #{参数名}
  1. resultMap ： 自定义结果集 ， 定义每一列的数据和 javabean 的映射
<!-- 自定义结果集
  type : 为那个javaBean自定义封装规则
  id: 唯一表示 给后面用的-->
<resultMap id="myEmployee" type="com.xixi.mybatis.dao.EmployeeDao">
    <!-- 指定主键列 ， 也可以用 result
                  column = "id" 指定那一列是主键列(数据库名字)
                  property = "" 指定 JavaBean 那个属性对应 id 这一列属性（JavaBean名字） -->
    <id property="id" column="id"/>
    <!-- 普通列 -->
    <result property="name" column="name"/>
    <result property="email" column="email"/>
    <result property="gender" column="gender"/>
</resultMap>

<!-- 返回list -->
<select id="getAllEmployee" resultMap="myEmployee">
    select * from t_employee
</select>
  • 通过 association 实现复杂类型映射
<resultMap id="myEmployee" type="com.xixi.mybatis.dao.EmployeeDao">
    <id property="id" column="id"/>
    <result property="name" column="name"/>
    <result property="email" column="email"/>
    <result property="gender" column="gender"/>
    <association property="类属性名" javaType="类"></association>
</resultMap>
  • 通过 collection 实现集合的封装
<resultMap id="myEmployee" type="com.xixi.mybatis.dao.EmployeeDao">
    <id property="id" column="id"/>
    <result property="name" column="name"/>
    <result property="email" column="email"/>
    <result property="gender" column="gender"/>
    <collection property="指定属性" ofType="类名">
        <id property="" column=""/>
        <result property="" column=""/>
    </collection>
</resultMap>

两种取值方式的区别

  1. #{名字} ：预编译方式 ， 安全的
  2. ${名字} ：用拼串方式 ， 不安全 ， sql 语句只有参数位置是支持预编译的 ， 如果想要表名动态传入 ， 这时候就可以用 ${}拼接了

一般使用 #{} 在不支持预编译的位置取值就使用 ${}

动态SQL

  • if 标签
<select id="getTeacherByTeacher" resultMap="teacherMap">
      select * from t_teacher where
      <!-- test 判断条件 OGNL 表达式-->
      <if test="id != null">id > #{id} and</if>
      <if test="name != null and name.equals(&quot;&quot;) ">teacherName like #{name} and</if>
      <if test="birth != null"> birth_date &lt; #{birth} </if>
  </select>
  • where 标签 代替sql中的 where ， 可以帮我们去除前面的and
<select id="getTeacherByTeacher" resultMap="teacherMap">
    select * from t_teacher
    <where>
    <if test="id != null"> and id > #{id}</if>
    <if test="name != null and name.equals(&quot;&quot;) ">and teacherName like #{name}</if>
    <if test="birth != null"> and birth_date &lt; #{birth} </if>
    </where>
</select>
  • trim 标签 ： 可以设置前缀后缀 ， 去除前后多余字符串
<select id="getTeacherByTeacher" resultMap="teacherMap">
    select * from t_teacher
    <!-- 截取字符串
    prefix 可以为我们下面sql整体添加前缀 一般写where，
    如果写在sql上，下面条件都不成立 ，就多了个 where

    prefixOverrides: 去除整体字符串前面多余的字符

    suffix : 为整体加后缀

    suffixOverrides ： 去除后面多余的字符串
    -->
    <trim prefix="where" prefixOverrides="and" >
    <if test="id != null"> and id > #{id}</if>
    <if test="name != null and name.equals(&quot;&quot;) ">and teacherName like #{name}</if>
    <if test="birth != null"> and birth_date &lt; #{birth} </if>
    </trim>
</select>
  • forch: 用来遍历集合
<select id="getTeacherIdIn" resultMap="teacherMap">
    select * from t_teacher where id in 
    <!-- 遍历集合 collection 默认名字得是类型
        item : 每次遍历到的元素的变量名
        open : 什么开始
        close : 什么结束
        index : 索引 , 如果遍历的是map index 就是 key
        separator : 每次遍历的分隔符 -->
    <foreach collection="list" item="item" separator="," open="(" close=")" index="index">
     #{item}
     </foreach>
</select>
  • set标签 ， 在update时使用可以自动去除 多余的, 号
<update id="updateTeacher" >
    update t_teacher
    <set>
        <if test="name != null"> teacherName = #{name}, </if>
        <if test="course != null"> class_name = #{course} , </if>
        <if test="address != null">address = #{address},</if>
    </set>
    <where>
        id = #{id}
    </where>
</update>

MyBatis 缓存

缓存：暂时的存储一些数据 ； 加快系统的查询速度

MyBatis缓存机制： Map ， 能保存查询出的一些数据

  • 一级缓存：线程级别的缓存；本地缓存 ， sqlSession级别的缓存；
    • 一级缓存失效的几种情况
      1. 一级缓存是 SqlSession级别的缓存 ， 如果有 2 个 SqlSession 会话 它们查询同一个数据不会走一级缓存。
      2. 同一个 SqlSession 查询 ， 不同参数 ，由于之前没查询过 ， 所以还会查询数据库
      3. 两个相同查询 中间执行了 增删改 操作 ， 增删改操作会清空缓存
      4. 可以调用 sqlSession.clearCache(); 手动清空缓存

每次查询， 如果一级缓存里面没有 ，就去发送新的 sql 查询数据

  • 二级缓存：全局范围的缓存；除当前线程能用外；SqlSession能用外其他也可以使用；namespace 级别的缓存

    • 二级缓存在 SQL Session 关闭或提交以后 ， 一级缓存的数据会放在二级缓存，如果要使用二级缓存需要在MyBatis配置文件中配置 , 注意配置顺序
    <settings>
    <setting name="cacheEnabled" value="true"/>
</settings>

  • 配置某个 dao.xml 文件 ， 让其使用 二级缓存

    <cache></cache>

  • 最后我们对应的 javaBean 还得支持序列化

  • 缓存的查询顺序

    1. 不会出现一级缓存和二级缓存有同一个数据
      1. 二级缓存中：一级缓存关闭就有了
      2. 一级缓存中：二级缓存没有数据，就会看一级缓存，一级缓存也没有，去查数据库放在一级缓存

    查询顺序 ： **2 –> 1 –> 库 **

  • 可以实现 Cache 接口自定义自己的缓存
