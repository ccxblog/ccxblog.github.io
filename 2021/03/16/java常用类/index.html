字符串

String

  • String 是声明为 final 的 ， 所以不可被继承

  • String实现了 Serializable 接口（可序列化接口）：所以String支持序列化

    • 实现了 Comparable 接口 ， 表示 String 可以比较大小

  • String 内部 定义了 final 的 char 类型数组 value 用于存储字符串数据 ， 所以数组不能被重新赋值，且数组的元素也不能被修改。

所以 String 代表一个不可变的字符序列 , 当对 String 对象进行赋值 或 进行连接操作时 需要重新指定内存区域赋值 ， 不能使用原有的 value 进行赋值 

// String 开头源码
public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence {
    /** The value is used for character storage. */
    private final char value[];

String的实例化方式

  1. 通过自变量定义的方式

  2. new 加构造器的方式

    import org.junit.Test;

public class Stringtest {
    @Test
    public void test(){
    // 自变量方式：此时 a , b 的数据 abc 声明在 方法区中的字符串常量池中
        
        String a = "abc";
        String b = "abc";

    //  new + 构造器的方式创建 ， 此时 c , d 存的是堆中的 new 出来的地址值
        String c = new String("abc");
        String d = new String("abc");
        
    }

}

执行语句String a = new String("abc")此时在内存中创建了 2 个对象 ， 一个是 new 出来的结构 ， 一个是 char[] 数组对应常量池的数据 “abc”。

String的拼接

常量与常量的拼接结果也在常量池中 ， 只要其中有一个变量 ，结果就在 堆 中 ， 如果调用 intern() 方法，返回的值是常量池的值

import org.junit.Test;

public class Stringtest {
    @Test
    public void test(){

        String a = "abc";
        String b = "def";

        String s = "abcdef";
        String c = "abc" + "def";

        String d = a + "def";
        String e = "abc" + b;
        String f = e.intern();

        System.out.println(c == s); // true
        System.out.println(c == d); // false
        System.out.println(c == e); // false
        System.out.println(c == f); // true
    }

}

StringBuffer

StringBuffer 代表可变长字符串 ， 可以对字符串进行增删，此时不会产生新对象，作为参数传递时，方法内部可以改变值。

简单源码分析

StringBuffer 的空参构造器 , char 数组初始化容量为 16

/**  
    * Constructs a string buffer with no characters in it and an
    * initial capacity of 16 characters.
    */
   public StringBuffer() {
       super(16);
   }

StringBuffer 带 String 参数的构造器 ， char 数组初始化容量为 16 + 传入的字符串长度

 public StringBuffer(String str) {
        super(str.length() + 16);
        append(str);
}

StringBuffer 扩容机制 ， 默认情况下扩容为原来的容量 * 2 + 2 , 同时将原来的元素赋值到新的数组中

private int newCapacity(int minCapacity) {
        // overflow-conscious code
        int newCapacity = (value.length << 1) + 2;
        if (newCapacity - minCapacity < 0) {
            newCapacity = minCapacity;
        }
        return (newCapacity <= 0 || MAX_ARRAY_SIZE - newCapacity < 0)
            ? hugeCapacity(minCapacity)
            : newCapacity;
}

String ， StringBuffer ， StringBuilder 的异同

String：不可变的字符序列,底层使用 final 修饰的 char 型数组存储

StringBuffer:可变的字符序列, 线程安全的，效率低 ， 使用 char 型数组存储

Stringbuilder:可变的字符序列,线程不安全的，效率高 ，和 StringBuffer 具有相同的父类 ， 底层也是 char 型数组存储

Date 类

Java.util.Date:

  • 构造器

    1. new Date() : 调用 toString 方法可以输出当前的 年，月，日 ， 时，分，秒，星期
    2. new Date(long date) : 传入一个毫秒数 ， 调用 toString 方法可以将传入的毫秒数转换为年，月，日 ， 时，分，秒，星期输出

  • 方法

    1. toString ：输出当前的 年，月，日 ， 时，分，秒，星期
    2. getTime： 获取当前Date对象对应的毫秒数
@Test
   public void test(){

java.text.SimpleDateFormat

  • 这是一个不与语言环境有关的方式来格式化和解析日期的具体类，它允许格式化 ：日期 -> 字符串，解析：字符串 -> 日期 

    @Test
    public void test1() throws ParseException {
        // 使用空参构造器创建
        SimpleDateFormat sdf = new SimpleDateFormat();
        Date date = new Date();
        // 通过format方法格式化 将date日期转换为 字符串
        String format = sdf.format(date);
        // 输出为 21-3-17 下午5:00
        System.out.println(format);

        // 将字符串解析为日期类型,因为字符串格式原因，会抛出异常
        // 这里格式默认为 通过format方法转换的格式
        Date date1 = sdf.parse(format);
        // 输出 Wed Mar 17 17:02:00 CST 2021
        System.out.println(date1);

        // ***************************
        // 使用带格式的构造器，可以自定义格式

        SimpleDateFormat sdf1 = new SimpleDateFormat("yyyy-mm-dd hh:mm:ss");
        String format1 = sdf1.format(date);
        // 按照上面自定义的格式输出 2021-07-17 05:07:15
        System.out.println(format1);
        // 传入的格式要和上面自定义的格式一样
        Date date2 = sdf1.parse("2021-07-17 05:07:15");
        // 输出 Sun Jan 17 05:07:15 CST 2021
        System.out.println(date2);
    }

Comparable

  • Sting 和 包装类 实现了 Comparable 接口 ， 重写了 ComparaTo(obj) 方法给出了比较两个对象大小的方法

  • 重写 ComparaTo 方法的规则：

    • 如果当前对象大于形参对象 obj 则返回正整数 ， 等于则返回 0 ， 否则返回负数

  • 对于自定义类 ， 我们需要实现Comparable 接口 然后 重写 ComparaTo(obj)方法

public class CompareTest implements Comparable{
    private String name;
    private double price;

    // 价格不一样按照价格排序 ， 否则按照名字排序
    @Override
    public int compareTo(Object o) {
        if (o instanceof CompareTest){
            CompareTest compareTest = (CompareTest)o;
            if(Double.compare(price , compareTest.price) != 0){
                return Double.compare(price , compareTest.price);
            }
            else return name.compareTo(compareTest.name);
        }
        throw new RuntimeException();
    }
}

Comparator

定制排序 , 可以不写在类里面 ， 在排序的时候重写方法

import org.junit.Test;

import java.util.Arrays;
import java.util.Comparator;

class Good{
    private String name;
    private double price;

    public String getName() {
        return name;
    }

    public double getPrice() {
        return price;
    }

    public Good() {}

    public Good(String name, double price) {
        this.name = name;
        this.price = price;
    }

    @Override
    public String toString() {
        return "Good{" +
                "name='" + name + '\'' +
                ", price=" + price +
                '}';
    }
}
public class ComparatorTest {
    @Test
    public void test() {
        Good[] gooda = new Good[3];
        gooda[0] = new Good("aa", 1.0);
        gooda[1] = new Good("cc" , 3.0);
        gooda[2] = new Good("bb" , 2.0);

        Arrays.sort(gooda, new Comparator<Good>() {
            @Override
            public int compare(Good o1, Good o2) {
                if(o1 instanceof Good && o2 instanceof Good){
                    Good a = (Good) o1;
                    Good b = (Good) o2;
                    if(Double.compare(a.getPrice() , b.getPrice()) != 0)
                        return Double.compare(a.getPrice() , b.getPrice());
                    else return a.getName().compareTo(b.getName());
                }
                throw new RuntimeException();
            }
        });

        System.out.println(Arrays.toString(gooda));
    }
}
