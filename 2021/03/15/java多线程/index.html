Java多线程

发表于 2021-03-15 更新于 2021-03-16 分类于 java
## 多线程

程序：是为了完成特定任务，用某种编程语言编写的一组指令集合。是一段静态的代码

进程：是程序的一次执行过程，或是正在运行的一个程序。是一个动态的过程。它有自身产生，存在，消亡的过程——————生命周期，进程是系统资源分配的最小单位。

线程：进程可进一步细化为线程，是一个程序内部的执行路径，线程作为调度和执行的最小单位，每个线程拥有独立的运行栈和程序计数器 PC，线程的切换开销小。

线程的生命周期

线程的生命周期关注两个概念：状态 ， 回调方法

  • 状态a –> 状态b：哪些方法执行了 （回调方法）
  • 某个方法主动调用 ， 导致状态a –> 状态b

多线程的创建

通过继承 Thread 类 然后 new 对象执行线程

package com.xixi.Thread;
// 1.创建一个类继承 Thread 类
class MyThread extends Thread{
// 2.重写 run 方法
    @Override
    public void run() {
        for (int i = 0; i < 1000 ; i++) {
            if(i % 2 == 0) System.out.println(i);
        }
    }
}

public class ThreadTest {
    public static void main(String[] args) {

        // 3.创建一个对象
        MyThread myThread = new MyThread();
        // 调用它的start()方法
        myThread.start(); // start()方法会创建一个线程 ， 然后调用run 方法

        for (int i = 0 ; i < 1000 ; i ++){
            if(i % 2 == 0) System.out.println(i + "++++++++");
        }
    }
}

实现 Callable 接口创建线程

  • 与Runnable相比 ， Callable功能更强大些
    • 相比 Runnable 的 run 方法 ， Callable 可以有返回值
    • 方法可以抛出异常
    • 支持泛型返回值
    • 需要借助FutrueTask类 ， 比如获取返回值
package com.xixi.java;

import java.util.concurrent.Callable;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.FutureTask;

// 1. 创建一个类实现 Callable
class NumThread implements Callable {

    // 2.重写 call 方法 ，线程要执行的操作写在 call 中
    @Override
    public Object call() throws Exception {
        int s = 0;
        for (int i = 0; i <= 100; i++) {
            if(i % 2 == 0)
                s += i;
        }
        return s;
    }
}

public class CallableTast {
    public static void main(String[] args) {
        // 3.new 一个 实现了Callable接口的对象
        NumThread numThread = new NumThread();
        // 4. 将实现了Callable接口的对象作为参数传递到 FutureTask 构造器中
        FutureTask future = new FutureTask(numThread);
        // 5.将 FutureTask 对象 作为参数传递到 Thread 的构造器中，并执行start方法
        new Thread(future).start();

        try {
            // 通过 FutureTask实例对象的 get 方法 获取 call方法 的返回值
            Object s = future.get();
            System.out.println(s);
        } catch (InterruptedException e) {
            e.printStackTrace();
        } catch (ExecutionException e) {
            e.printStackTrace();
        }
    }
}

使用线程池创建线程

  • 如果经常创建和销毁进程，会影响系统的性能，所以可以提前创建好多个线程，然后放进线程池中，要用的时候就直接从线程池拿，用完后再放进线程池中。最终实现重复利用。
package com.xixi.java;

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

// 1，创建线程对象 ， 并重写 run 方法
class MyThreads implements Runnable{

    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " aaa");
    }
}

class MyThreadtow implements Runnable{

    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " bbb");
    }
}


public class Pool {
    public static void main(String[] args) {
        // 2.创建池子 , newFixedThreadPool 参数为线程池的大小
        ExecutorService executorService = Executors.newFixedThreadPool(10);
        // execute  适合用于 Runnable
        // submit 适合用于 Callable
        // 执行指定线程的操作 ， 需要提供一个实现Runnable 或 Callable 接口的对象
        executorService.execute(new MyThreads());
        executorService.execute(new MyThreadtow());

        // 关闭连接池
        executorService.shutdown();



    }
}

Thread的常见方法

  1. start(): 启动当前线程 ， 然后调用run()方法
  2. run() :通常要重写 Thread中的run()方法，创建线程后要执行的代码写在run()中
  3. currentThread():静态方法，返回执行当前代码的线程
  4. getName() :返回当前线程的名字
  5. setName():设置当前线程的名字
  6. yield():释放当前CPU的执行权(有可能下次又是分配到当前线程)
  7. join() :在线程a调用线程b的join方法 , 此时线程a将进入阻塞状态，直到线程b执行完后，线程a才结束阻塞状态
  8. sleep(long millitime): 让当前线程“睡眠”指定的 millitime 毫秒数 ， 在指定的毫秒时间内，当前线程处于阻塞状态
  9. isAlive(): 判断当前线程是否存活，线程的run方法执行完了当前线程就“死”了 
package com.xixi.Thread;

class MyThreadtesta extends Thread{

    @Override
    public void run() {

        for (int i = 0; i < 100; i++) {
            if(i % 2 == 0) {
                try {
                    sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println(Thread.currentThread().getName() + ":" + i);
            }
            if(i % 20 == 0) yield();
        }
    }
}

public class ThreadMethodTest {
    public static void main(String[] args) {
        MyThreadtesta a = new MyThreadtesta();
        a.start();

        for (int i = 0; i < 100; i++) {
            if(i % 2 == 0)
                System.out.println(Thread.currentThread().getName() + ':' + i);
            if(i == 20){
                try {
                    a.join();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
        System.out.println(a.isAlive());
    }
}

  2. MIN_PRIORITY: 1 最小优先级是1
  3. NORM_PRIORITY: 5默认优先级是5

如何获取和设置线程的优先级

  • getPriority()： 获取线程的优先级
  • setPriority()：设置线程的优先级

高优先级的线程抢占cpu执行权成功的概率要比低优先级的线程抢占cpu执行权的概率要高

线程安全

在 Java 中，我们通过同步机制，来解决线程安全问题

  • 同步代码块 
synchronized(同步监视器){
    // 需要被同步的代码 ， 即需要操作共享数据的代码
    // 同步监视器：俗称 锁 ， 任何一个类的对象都能重当 锁 ， 并要求多个线程公用同一把锁
}

package com.xixi.java;

class Windows implements Runnable{

    Object obj = new Object();
    private int ticket = 100;
    @Override
    public void run() {

        while(true){
            synchronized (obj) {
                if (ticket > 0) {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                 System.out.println(Thread.currentThread().getName() + " " + ticket);
                    ticket--;
                }
                else break;
            }
        }
    }
}

public class WindowsThread {
    public static void main(String[] args) {

        Windows windows = new Windows();
        Thread t1 = new Thread(windows);
        Thread t2 = new Thread(windows);
        Thread t3 = new Thread(windows);

        t1.start();
        t2.start();
        t3.start();

    }
}

解决线程安全的方式 3 Lock锁

package com.xixi.java;

import java.util.concurrent.locks.ReentrantLock;

class Windowsa implements Runnable {

    private int ticket = 100;
    // 核心 ， 如果为 true 表示公平的，先来的线程先获取资源 
    private ReentrantLock lock = new ReentrantLock(true);

    @Override
    public void run() {
        while(true)
        {

            try {
                lock.lock(); // 手动加锁
                if(ticket > 0) {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }

                  System.out.println(Thread.currentThread().getName() + ":" + ticket);
                    ticket--;
                }
                else break;
            }finally {
                lock.unlock(); // 释放锁
            }

        }
    }
}

public class LockTest {
    public static void main(String[] args) {
        Windowsa a = new Windowsa();
        Thread t1 = new Thread(a);
        Thread t2 = new Thread(a);
        Thread t3 = new Thread(a);
        t1.start();
        t2.start();
        t3.start();
    }
}

synchronized 与 Lock 锁的异同

相同：都可以用来解决线程安全问题

不同：synchronized 机制在执行完相应同步代码后，会自动释放同步监视器

​ Lock 需要手动开启 和 手动释放

进程的通信

wait(): 调用 wait() 方法的线程进入阻塞状态 ， 并释放锁。

notify(): 唤醒一个被wait()阻塞的线程

notifyAll(): 唤醒所有被wait()阻塞的线程

wait() , notify notifyAll 这 3 个方法是定义在 Object类中的， 三个方法必须使用在同步代码块或同步方法中，并且它们的调用者必须是同步代码块或同步方法中的同步监视器

package com.xixi.java;

class Number implements Runnable{

    private int a = 1;
    @Override
    public void run() {
        while(true)
        {
            synchronized (this) {
                notify();
                if (a > 100) break;
                System.out.println(Thread.currentThread().getName() + ":" + a);
                a++;
                try {
                    wait();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

public class CommunitationTest  {
    public static void main(String[] args) {
        Number number = new Number();
        Thread t1 = new Thread(number);
        Thread t2 = new Thread(number);
        
        t1.start();
        t2.start();
    }
}

sleep 和 wait 的异同

  • 相同点： 一旦执行方法 ， 线程都会进入阻塞状态
  • 不同点：
    • 两个方法声明的位置不一样：Thread类中声明的 sleep() , Object 类中声明的 wait()
    • 调用的位置不同 ， sleep()可以在任何需要的场景调用 ， wait()必须在使用同步代码块/方法区的内部调用
    • 关于是否释放同步监视器的问题：如果两个方法都使用在同步代码块/方法区，sleep不会释放锁 ， wait会释放锁
